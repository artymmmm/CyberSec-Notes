## Анализ образов памяти
- Volatility Framework - анализ дампов оперативной памяти ([версия 3.0](https://github.com/volatilityfoundation/volatility3), [версия 2.6](https://github.com/volatilityfoundation/volatility)): 
	- Получения профиля дампа (ОС, версия): `python vol.py -f <image> imageinfo - получение профиля (ОС, версия)`
	- Дамп процессов (`pslist`, `pstree`, `psscan`): `python vol.py -f <image> pslist —profile=<profile>`
	- Дамп памяти процесса: `python vol.py -f <file> —profile=<profile> memdump -p <PID>`
	- Сетевые подключения: `python vol.py -f <image> —profile=<profile> connections`
	- Команды, которые были выполнены через CMD: `python vol.py -f <image> —profile=<profile> cmdscan`
- [FTK Imager](https://www.exterro.com/digital-forensics-software/ftk-imager)
- [R-Studio](https://www.r-studio.com/): восстановление файлов, обход файловой системы
- [Elcomsoft](https://www.elcomsoft.ru/): получение ключей из образа, расшифровка образа
- aeskeyfind: поиск ключей AES в образе памяти (https://medium.com/@Frogjump/aeskeyfind-in-kali-linux-72ba6a8ea2fd), BitLocker использует AES
