Раскрытие/утечка информации - это непреднамеренное разглашение веб-сайтом конфиденциальной информации своим пользователям. Такая информация может включать: 
- данные о других пользователях (логины, email, ПДн)
- конфиденциальные коммерческие или деловые данные компании
- технические сведения о веб-сайте и его инфраструктуре
## Способы тестирования на раскрытие информации
- Фаззинг (Burp Intruder, BApp store extension "Logger++" позволяет фильтровать запросы и ответы из всех инструментов Burp Suite)
- Burp Scanner
- Burp's engagement tools: Search (поиск в HTTP-запросе), Find comments (Target->Sitemap, поиск комментариев), Discover content (поиск каталогов и файлов)
- Создание информативных ответов: атакующий может сделать специальный запрос с целью получения дополнительной информации из выводимых ошибок (stack trace или debug)
## Источники раскрытия информации
- Файлы для веб-краулеров: `/robots.txt`, `/sitemap.xml`
- Списки каталогов: веб-сервер может быть настроен так, что он автоматически отображает содержимое каталогов, в которых отсутствует индексная страница (`index.html`)
- Комментарии разработчиков в коде
- Сообщения об ошибках
- Отладочные данные
- Страница пользователя (через ошибки в контроле доступа, IDOR)
- Раскрытие исходного кода через файлы бэкапа (например, текстовые редакторы создают копии файлов, в которых работают, можно попытаться получить к ним доступ: `~file.php`)
- Раскрытие информации из-за небезопасной конфигурации (например, метод HTTP TRACE предназначен для диагностических целей. Если он включен, веб-сервер будет отвечать на запросы, использующие метод TRACE, отображая в ответе точный текст полученного запроса, но также может содержать дополнительную информацию в виде дополнительных заголовков)
- Контроль версий (Git): на сайтах может быть каталог `.git`
